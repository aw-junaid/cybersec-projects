# ICS Lab Rules & Ethics

## REQUIRED: Before Starting

1. Obtain written authorization from security lead
2. Physically disconnect from production networks  
3. Verify air-gap with `check_isolated.py`
4. Document all testing activities

## Permitted Activities

- Protocol analysis and reverse engineering
- Safe exploit testing on simulated endpoints
- IDS signature development and testing
- Forensic analysis of captured traffic

## Prohibited Activities  

- Connecting to real ICS devices
- Denial-of-service testing on production systems
- Weaponization of exploits for unauthorized use
- Removing lab materials from controlled environment

## Incident Reporting

Immediately report any:
- Accidental connection to production networks
- Safety control failures
- Unauthorized access attempts
